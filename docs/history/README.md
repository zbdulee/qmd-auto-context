# Historical Design Records

This directory contains historical implementation plans and design
specifications. They preserve decision context but are not authoritative for
the current product behavior.

For current behavior, use [README.md](../../README.md),
[docs/settings.md](../settings.md), [docs/architecture.md](../architecture.md),
and the current code and tests. Historical documents may describe superseded
config names, host assumptions, or implementation states.
